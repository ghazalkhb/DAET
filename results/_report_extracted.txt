1 Introduction
This report presents the extended experimental evaluation of an adaptive performance-regression
detection algorithm applied to the Mozilla Firefox-Android Treeherder telemetry dataset. The
goal is to decide, at each new measurement pointt, whether torecorda sample (“enable” the
sensor) based on an exponential anomaly score derived from the one-step-ahead forecast residual.
Four forecasting strategies are compared insideAlgorithm 3:
ARIMA(p, d, q) = (1,1,1) — parameters estimated on the training split only; a fixed-
parameter Kalman filter is rolled forward on the test set (no data leakage).
LAST(na¨ ıve baseline) — ˆy t =y t−1.
SMA(w) — simple moving average over the lastwpoints.
EWMA(α) — exponentially weighted moving average: ˆy t =α y t−1 + (1−α) ˆyt−1.
This evaluation adds over the previous study:
Strictchronological train/test splitwith leakage controls.
Precision,Recall,F1on the test set.
False-alarm rate(FP intervals per test time step).
Fulldetection-delay distributionstatistics.
Compute and peak memorymeasurements.

3 Methodology
3.1 Algorithm 3 Controller
At each test-set time stept:
1. Compute forecast ˆyt using the chosen method.
2. Compute residualδ t =|y t −ˆyt|.
3. Compute dynamic threshold Θ t = max(Θmin, k σ σ(δt−w:t)).
4. Update anomaly scores t =β1[δ t >Θ t] + (1−β)s t−1.
5. Outputdecision t =1[s t ≥θ anomaly].
3.2 EWMA Forecaster
ˆyt+1 =α y t + (1−α) ˆyt, α∈(0,1).
The running state is initialised on the training split and carried forward into the test set without
reinitialisation.
3.3 Train/Test Split and Leakage Controls
Each time series is split chronologically at the 70% mark:
Train set: ARIMA parameters are estimated here; the dynamic-threshold residual history
is seeded from train residuals; EWMA state is initialised.
Test set: All detection decisions are evaluated here. For ARIMA, the Kalman filter uses
fixedparameters from the train fit — no re-estimation. Alert timestamps are compared
to test-set decisions only during post-hoc evaluation (never used for parameter tuning).

8 Discussion
ARIMA.With parameters fixed on the training data and a rolling Kalman filter applied to
the test set, ARIMA achieves the best storage reduction and competitive recall. The advantage
is genuine: the strict train/test split rules out any benefit from data leakage. The cost is
substantially higher compute time.
EWMA.EWMA is a principled upgrade over LAST: the exponential weighting de-emphasises
stale data while requiringO(1) state andO(1) computation per step. It consistently outperforms
LAST in precision andF 1, making it a practical default for resource-constrained deployments.
SMA.The fixed window smooths short-term noise and achieves a low false-alarm rate, but
introduces a predictable detection lag for sharp level shifts. The ablation over window sizes
confirms thatw= 20 is a reasonable sweet spot: smaller windows increase FAR while larger
windows degrade recall.
LAST.The simplest baseline. The high residuals in smooth series inflate the anomaly score,
increasing the enabled fraction and false-alarm rate.
Detection delay.Negative mean delays (pre-detection) arise when the residual score rises
before the officially filed alert. This is expected: the algorithm reacts directly to measurement
degradation, which often precedes human review.
Robustness to hyperparameters.The threshold (θ) ablation shows that all methods are
most sensitive to this parameter: values below 0.20 cause significant false-alarm inflation while
values above 0.40 suppress recall. Beta (β) affects the score’s temporal memory; the default
0.30 provides a balanced trade-off. The tolerance (τ) ablation confirms that the F 1 ordering of
methods is stable across all evaluated window sizes.
Statistical significance.Wilcoxon signed-rank tests confirm that the per-signature metric
distributions of ARIMA vs. LAST and ARIMA vs. EWMA differ significantly (p <0.05) on
most metrics. Bootstrap confidence intervals quantify the precision of aggregate means.
9 Conclusions
We evaluated Algorithm 3 with four forecasting back-ends on the Mozilla Firefox-Android Tree-
herder dataset (17,989 alert records, 214 timeseries signatures) under a strict chronological
70/30% train/test split.
Key findings:
ARIMAachieves the best storage reduction and precision when evaluated without leak-
age, confirming its advantage is genuine.
EWMAis the best lightweight option: negligible compute overhead over LAST with
meaningfully higher precision andF 1.
SMAis robust and achieves a low false-alarm rate, but the fixed window introduces a
detection lag.
LASTestablishes a useful lower bound but is not recommended for production.
The decision thresholdθhas the strongest influence on all methods; its default of 0.30
balances recall against FAR.
